# Contributing to This Project

Thank you for your interest in contributing! This project is maintained by **Royalda**.

---

## Before You Contribute

All contributors must sign our **Contributor License Agreement (CLA)** before a pull request can be merged.

This is a one-time step — once signed, it covers all future contributions across Royalda repositories.

When you open a pull request, a bot will automatically ask you to sign the CLA via a comment. It takes about 30 seconds.

**[Read the CLA →](.github/CLA.md)**

---

## How to Contribute

### Reporting Bugs
- Open an issue with a clear title and description
- Include steps to reproduce and expected vs actual behaviour
- Attach screenshots or logs if relevant

### Suggesting Features
- Open an issue tagged `enhancement`
- Describe the use case and why it belongs in the project

### Submitting a Pull Request
1. Fork the repository
2. Create a feature branch: `git checkout -b feat/your-feature`
3. Make your changes and commit with a clear message
4. Push to your fork and open a PR against `main` (or `master`)
5. Sign the CLA when prompted by the bot
6. Wait for review

### Commit Message Style
```
feat: add user export endpoint
fix: resolve session timeout on mobile
docs: update API reference
chore: bump next to 16.2.1
```

## Security Issues

**Do not open a public issue for security vulnerabilities.**
Report them privately at **security@royalda.com** — see [SECURITY.md](SECURITY.md).

---

## Code Style

- Follow the existing patterns in the codebase
- Run `npm run lint` before submitting
- No commented-out code or debug logs in PRs
